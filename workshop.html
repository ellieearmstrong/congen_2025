Getting started

In this module we’ll walk through hands-on steps for genome assembly and basic quality checks. We’ll work inside the ConGen environment (for example, via an RStudio Server session) and most of what you do will be through a Terminal tab.

If you haven’t already, connect to the ConGen server and open RStudio in your browser. Make sure you can see the Console and the Terminal; if your interface looks very different from what the instructors are showing, please ask for help.

Throughout the workshop, shell commands will be presented in fixed-width blocks like this:

this_is_an_example_command --with-some-options

After some commands, we may break down important flags or arguments so you can see what each piece does, rather than treating the command line as a black box. Those explanations are there to help; if they feel like too much detail, you can skim past them and focus on the commands and outputs.

Tip — Help menus

Most command-line tools provide a built-in help menu. You can usually view it with:

<program> -h # or <program> <sub-command> -h

For many standard Linux commands, you can also use the manual pages:

man <command>

The man viewer can be navigated with the arrow keys, and you can quit by pressing q. If you’re ever unsure about what a command or option does, try the help menu or manual page first.

Tip — Looking at your data

A crucial part of bioinformatics is simply looking at your data and intermediate outputs. We’ll pause at several points to inspect what a command produced, often showing a short snippet of the output so you can compare what you see on your screen with the walkthrough.

Input files & output prep

Copying input data

For this workshop we’ll primarily work with sequence data in FASTA and FASTQ formats. The instructor has placed a set of input files on the server in a shared directory (adjust this path to match your actual setup). We’ll begin by copying that directory into our own home directory so we can work on an independent copy.

First, move to your home directory:

cd ~

Here cd is the “change directory” command, and ~ is a shortcut that means “my home directory”.

Now copy the workshop data directory into your home directory:

cp -r instructor_materials/Gregg_Thomas/congen-assembly/ .

In this command, cp is the copy command, -r tells it to copy directories recursively, the long path indicates the folder you’re copying from, and . means “copy it here, keeping the same name”.

Change into the new directory:

cd congen-assembly

Then check what’s inside:

ls

You should see several subdirectories, for example:

  • dmel-3R-reference/ – a reference sequence and indices for one chromosome from a comparison species.
  • dpse-chr2-reads/ – example short-read data for the assembly exercise.
  • expected-outputs/ – pre-computed outputs for each step, in case something fails or takes too long on the fly.
  • scripts/ – helper scripts used in some of the analyses.
Tip — Pre-generated outputs

If you get stuck on a command or your job is taking too long, you can often continue to the next step by using the corresponding file from expected-outputs/ instead of the file you were supposed to create. When in doubt, ask an instructor which file to use.

Preparing output directories

It’s useful to plan ahead for where your outputs will go so that your working directory stays organized. For this module we’ll generate assemblies, read mappings, and alignments, so we’ll create a separate directory for each.

mkdir alignments mkdir assemblies mkdir mapped-reads

Here mkdir creates a new directory with the specified name. We’ll use these folders to store alignments, assemblies, and mapped reads, respectively.

Some tools require that you create an output directory ahead of time. For example, FastQC expects a directory where it can place its reports:

mkdir fastqc-output

Other programs will create their own subdirectories automatically. At this point you should have a clean structure ready to hold the results of each step of the assembly and mapping workflow.
