Workshop

This page walks you through the hands-on assembly module for ConGen2025. The sections below expand to show more detail for each stage of the exercise.

Use these as a reference while you work through the activities during the course. You can paste in the exact commands and SLURM scripts you want participants to use.

Start ▶

This section covers logging in, finding your data, and orienting yourself to the directory structure you will use during the workshop.

  1. Log into the cluster or environment specified by your instructors.
  2. Navigate to the working directory for the assembly exercises.
  3. List the contents and identify the read files and any reference materials.
# Example: connect to the cluster and move to your working directory ssh your_user@your_cluster cd /path/to/congen2025/assembly/ ls

You can customize this area with your exact paths, project structure, and any environment setup steps (modules, conda environments, etc.).

Reads ▶

In this section, you will explore the read data you’ll be assembling: file formats, coverage, and basic quality control.

  1. Identify read types (short vs. long reads, paired-end vs. single).
  2. Check the number of reads and approximate coverage for the target genome.
  3. Run quick QC (e.g. FastQC or summary statistics) to assess read quality.
# Example: inspect read files ls data/ # Example: run a basic read summary (replace with your real commands) zcat data/sample_R1.fastq.gz | head -n 8 # Example: placeholder for FastQC or similar # fastqc data/sample_R1.fastq.gz data/sample_R2.fastq.gz -o qc/

Replace the example commands above with the tools and parameters you want to highlight (e.g. FastQC, seqtk, custom scripts, or coverage calculators).

Assembly ▶

Here you will run a genome assembler, inspect the resulting assembly, and (optionally) map reads back to the assembly to evaluate its quality.

  1. Choose an assembler appropriate for your read type.
  2. Run the assembler with a small set of core parameters.
  3. Summarize the resulting assembly (N50, total size, number of contigs/scaffolds).
  4. Optionally map reads back to the assembly and examine coverage and potential issues.
# Example: run an assembler (replace with your actual tool and parameters) # spades.py -1 data/sample_R1.fastq.gz -2 data/sample_R2.fastq.gz -o assembly/ # Example: summarize assembly # python scripts/assembly_stats.py assembly/contigs.fasta # Example: map reads back (placeholder) # bwa index assembly/contigs.fasta # bwa mem assembly/contigs.fasta data/sample_R1.fastq.gz data/sample_R2.fastq.gz | samtools sort -o mapped.bam # samtools index mapped.bam
